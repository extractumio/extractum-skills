# Domain Specific Skills

Specialized skills tailored for specific industries, frameworks, or use cases.

## About Domain Specific Skills

Unlike general-purpose skills, these are designed for particular:
- **Industries**: Finance, Healthcare, E-commerce, etc.
- **Frameworks**: React, Django, Spring Boot, etc.
- **Technologies**: Kubernetes, AWS, PostgreSQL, etc.
- **Workflows**: Data science, ML ops, API development, etc.

## Contributing Domain Specific Skills

To add a new domain-specific skill:

1. Create a directory: `domain-specific/your-skill-name/`
2. Add `SKILL.md` following the standard template
3. Include domain-specific context and examples
4. Document all dependencies and prerequisites
5. Update this README to list the new skill
6. Submit a pull request

### Criteria for Domain Specific Skills

- ✅ Focused on a specific domain, framework, or technology
- ✅ Includes domain-specific best practices
- ✅ Clearly documents prerequisites and dependencies
- ✅ Provides domain-relevant examples
- ✅ Tested in the target domain environment

### Naming Convention

Use descriptive names that indicate the domain:

- `react-component-generator` (not just `component-generator`)
- `kubernetes-helm-deploy` (not just `deploy`)
- `django-api-scaffold` (not just `api-scaffold`)
